# 服务异常与客户通知规范

## 适用范围

当客户反馈接口超时、页面无法访问、核心功能不可用、任务大量失败或生产环境受阻时，客服需要按照服务异常流程处理。

## 处理步骤

1. 记录客户影响范围，包括工作区、发生时间、错误码、接口路径和截图。
2. 查询系统状态页和内部告警。如果已有公开故障公告，应同步公告链接和当前处理进度。
3. 如果影响多个客户或核心流程，应升级给值班工程师和客服主管。
4. 不要在没有工程确认前承诺具体恢复时间，可以说明团队正在排查，并会持续同步进展。

## 回复要点

- 先承认客户受到影响。
- 说明正在排查或已升级。
- 请求客户补充错误时间、接口路径、截图和影响范围。
- 如果已有临时绕行方案，应明确说明适用条件。
